=================== работа с коммандной строкой =============================


		 *Формат комманды определяется тем,
		  применяется она к DataObject или Datafield

 Комманда:


  "create" 	 - вызов модуля создания объекта данных,
		  затем инициализация объекта данных.


	Ключи комманды create:
	    -o 	- выполнение комманды по отношению к DataObject
	    -f  - выполнение комманды по отношению к Datafield



  "field"	 - комманда, вводимая после комманды create.
		   Определяет имя и значение 
		   нового поля Datafield в DataObject.

  "done"	 - комманда, вводимая после комманды create.
		   Сообщает программе, о завершении создания
		   объекта DataObject.


	 	1) Пример создания DataObject:
			
			----------------------Создание объекта с указанием полей:

			create -o name    	;создать объект с именем name 
	 	  	field f_name value	;создание в этом объекте поля с именем "f_name" и значением (value)
	 	  	...
	 	  	...
	   		field f_name value
	 	  	done			;сообщение программе завершить работу с объектом

			----------------------Создание пустого именованного объекта:

		 	create -o name		;создать объект с именем name 
			done			;сообщение программе завершить работу с объектом
		
			------------------


		2) Пример создания Datafield в существующем объекте:

	   		create -f object_name f_name f_value	;создать поле с именем "f_name" и значением (value) 
								;в объекте "object_name"

			------------------
		


  "delete"	 - вызов модуля освобождения памяти объекта данных.

	Ключи комманды delete:
	    -o 	- выполнение комманды по отношению к DataObject
	    -f  - выполнение комманды по отношению к Datafield

		
		1) Пример удаления существующего DataObject:

			-----------------------Удаление объекта:

			delete -o object_name			;удалить объект с именем "object_name"

			------------------

		2) Пример удаления поля в существующем DataObject:

			-----------------------Удаление поля:

			delete -f object_name field_name	;удалить объект с именем "object_name"

			------------------



=================== структура данных программы =============================

       Структура данных программы основывается на трех объектах:


1| "Datafield" - Универсальная ячейка хранения объекта данных,
		   которая представляет собой указатель на объект данных.
		   
2| "DataObject"  - Динамический массив, хранящий некоторое количество
		   объектов Datafield.

3| "DataClass"   - Динамический массив, хранящий некоторое количество
		   объектов DataObject.



=================  модуль сохранения данных сессии  =========================
		  и восстановления данных из памяти



1| Сохранение данных      - для сохранения данных используется
   программы в память       запись символьных и числовых значений
			    в текстовый файл,
			    с указанием условных характеристик объекта.

2| Восстановление данных  - для восстановления данных используется
   программы из памяти      чтение символьных и числовых значений,
			    а также характеристик объектов,
			    из текстового файла.



=================  модуль графического интерфейса  =========================


1| Экран справки          - выводится при начале работы с программой,
			    а  так  же  при   вводе  комманды  "help"
			    в коммандную строку программы.

2| Коммандная строка      - после отрисовки, программа ждет ввода 
   программы		    комманды по типу:
			    "comm_name key_t".

3| Экран вывода		  - вывод содержания структуры данных,
   текстовой информации     ожидание комманды от пользователя.
